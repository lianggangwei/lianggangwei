# 远程桌面设置完整指南

## ⚠️ 重要安全警告

**从外网开放远程桌面非常危险！请务必阅读以下安全建议。**

---

## 📋 完整设置步骤

### 第一步：运行设置脚本（需要管理员权限）

1. 找到文件：`setup_remote_desktop.ps1`
2. **右键点击** → 选择 **"以管理员身份运行"**
3. 脚本会自动：
   - ✅ 启用远程桌面
   - ✅ 启用网络级身份验证（更安全）
   - ✅ 配置Windows防火墙

---

### 第二步：设置用户密码（必须手动完成）

我无法自动帮您设置密码，请按以下步骤操作：

1. 按 `Win + X` 键，选择 **"Windows PowerShell (管理员)"** 或 **"命令提示符(管理员)"**

2. 输入以下命令并回车：
   ```cmd
   net user Administrator lianggangwei123
   ```
   （如果您使用的是其他用户名，把 `Administrator` 换成您的用户名）

3. 或者通过图形界面设置：
   - 按 `Win + I` 打开设置
   - 选择 **"账户"** → **"登录选项"**
   - 点击 **"密码"** → **"添加"**
   - 设置密码为：`lianggangwei123`

---

### 第三步：获取您的电脑信息

#### 获取局域网IP地址：
1. 按 `Win + R`，输入 `cmd`，回车
2. 输入：`ipconfig`
3. 找到 **"IPv4 地址"**，例如：`192.168.1.100`

#### 获取公网IP地址（外网访问用）：
1. 打开浏览器，访问：`https://www.whatismyip.com/`
2. 记下显示的公网IP地址

---

### 第四步：路由器端口转发（外网访问必须！）

如果要从外网访问，必须在路由器上设置端口转发：

1. 打开浏览器，访问路由器管理页面（通常是 `192.168.1.1` 或 `192.168.0.1`）
2. 登录路由器（查看路由器底部的用户名和密码）
3. 找到 **"端口转发"** 或 **"虚拟服务器"** 或 **"NAT转发"** 设置
4. 添加一条规则：
   - **外部端口**：`3389`
   - **内部端口**：`3389`
   - **内部IP地址**：您电脑的局域网IP（例如：`192.168.1.100`）
   - **协议**：TCP
5. 保存设置

---

## 🖥️ 如何连接远程桌面

### 从局域网内连接：
1. 按 `Win + R`，输入 `mstsc`，回车
2. 在 **"计算机"** 栏输入：`您的局域网IP:3389`
   - 例如：`192.168.1.100:3389`
3. 点击 **"连接"**
4. 输入用户名和密码：
   - 用户名：`Administrator`（或您的用户名）
   - 密码：`lianggangwei123`

### 从外网连接：
1. 按 `Win + R`，输入 `mstsc`，回车
2. 在 **"计算机"** 栏输入：`您的公网IP:3389`
   - 例如：`123.45.67.89:3389`
3. 点击 **"连接"**
4. 输入用户名和密码

## 🔒 安全增强建议（必读！）

### 1. 密码安全（必须！）
- ✅ 当前密码：`lianggangwei123` - 这个密码不够强
- 🔴 建议改为：至少12位，包含大小写字母、数字、特殊符号
- 例如：`Liang@GangWei2026!`

### 2. 网络安全
- ✅ 已启用网络级身份验证（NLA）- 更安全
- ⚠️ 建议只在信任的网络环境下使用外网访问
- 🔴 不要在公共WiFi下使用远程桌面

### 3. 其他安全措施
- 定期检查Windows更新
- 安装杀毒软件并保持更新
- 考虑使用VPN而不是直接开放3389端口
- 定期查看远程桌面登录日志

### 4. 如果不需要外网访问
- 只在局域网内使用是最安全的
- 不需要配置路由器端口转发
- 防火墙只允许局域网内的连接

## 🛑 如果遇到问题

### 问题1：连接失败
- 检查防火墙是否允许远程桌面
- 确认远程桌面已启用
- 检查网络连接

### 问题2：密码错误
- 确认密码已正确设置
- 尝试重新设置密码

### 问题3：外网无法连接
- 确认路由器端口转发已设置
- 检查公网IP是否正确
- 确认ISP没有封锁3389端口
